Getting Started

Installation

Install the enhanced anime.js fork via CDN or npm:

// Via CDN
<script src="https://cdn.jsdelivr.net/npm/animejs-fork@latest/lib/anime.min.js"></script>

// Via npm
npm install animejs-fork

// ES6 import
import anime from 'animejs-fork';

Basic Usage

Create your first animation with enhanced precision:

anime({
  targets: '.my-element',
  translateX: 250,
  rotate: '1turn',
  backgroundColor: '#ff4757',
  duration: 800,
  easing: 'easeInOutQuad'
});

3D Transforms

Enhanced 3D capabilities with hardware acceleration:

anime({
  targets: '.cube',
  rotateX: '360deg',
  rotateY: '360deg',
  rotateZ: '180deg',
  translateZ: 100,
  perspective: 1000,
  duration: 2000,
  easing: 'easeInOutCubic'
});

Timeline Control

Create complex animation sequences with precise timing:

const tl = anime.timeline({
  easing: 'easeOutExpo',
  duration: 750
});

tl
.add({
  targets: '.element1',
  translateX: 250
})
.add({
  targets: '.element2',
  translateY: -100
}, '-=500') // Start 500ms before previous animation ends
.add({
  targets: '.element3',
  scale: [0, 1]
});

Stagger Animations

Animate multiple elements with intelligent staggering:

anime({
  targets: '.stagger-element',
  translateY: [-40, 0],
  opacity: [0, 1],
  delay: anime.stagger(100, {
    grid: [3, 3],
    from: 'center'
  }),
  duration: 600
});

API Reference

Core Methods

// Basic animation
anime(options)

// Timeline
anime.timeline(options)

// Stagger utility
anime.stagger(value, options)

// Random utility
anime.random(min, max)

// Path utilities (enhanced)
anime.path(pathElement)
anime.setDashoffset(pathElement)

Enhanced Easing Functions

Extended easing library with physics-based animations:

// Custom cubic-bezier
'cubicBezier(x1, y1, x2, y2)'

Callbacks & Events

anime({
  targets: '.element',
  translateX: 250,
  begin: function(anim) {
    console.log('Animation started');
  },
  update: function(anim) {
    console.log('Progress:', anim.progress);
  },
  complete: function(anim) {
    console.log('Animation completed');
  }
});

Performance Optimization

Best Practices

  • Use transform properties for better performance
  • Prefer opacity over visibility
  • Use will-change CSS property for complex animations
  • Batch DOM reads and writes
  • Use hardware acceleration with 3D transforms
// Optimized animation
anime({
  targets: '.element',
  translateX: 250, // Uses transform (GPU accelerated)
  opacity: [0, 1], // Efficient opacity change
  duration: 800,
  easing: 'easeOutQuad'
});

// CSS optimization
.element {
  will-change: transform, opacity;
  transform: translateZ(0); // Force hardware acceleration
}
